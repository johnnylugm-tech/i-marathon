## 一、台灣市場到底有沒有真實需求？—— 有，但要先看清楚棋盤

### 1.1 市場不是藍海，是已有競爭的紅海

這是第一個重要的認知校正：**i-marathon 想要解決的問題，在台灣已經被一群廠商服務了 10 年以上**。

| 服務商 | 規模 / 定位 | 強項 | 弱點（跑者真實痛點） |
|---|---|---|---|
| **AllSports（創星影像）** | 日本 Photocreate 在台合資，全台最大規模；單場臺北馬 68 萬張照片 | 全台最大攝影師網絡 | 「找照片」痛苦（68 萬張中找你）、單張 NT$80-200、肖像權爭議 |
| **Zsport** | 商業攝影平台；單場環縣自行車 39 萬張 | 單場照片量大 | 同上付費牆問題 |
| **Sportag 運動標籤** | FB 34,749 讚、主打 AI 人臉+號碼布 | AI 辨識 | 找得到 ≠ 自動推播、須自拍照上傳 |
| **RaceShot 運動拍檔** | AI 辨識 + 號碼布/服裝/GPX 多維搜尋 | 搜尋體驗佳 | 同樣不推播，付費下載 |
| **Phomi 瘋迷** | 7-11/全家/萊爾富機台列印明信片 | **實體紀念價值** | 無自動推播 |
| **好拍 gooodshot** | 老牌賽事照片平台 | — | 同上 |

**Threads 用戶 @deng1994 在「身為路跑選手，一定要知道賽道照片怎麼找！」一文，列出 7 個平台才湊齊賽事照片抓取區**——這就是「找照片馬拉松」最生動的證據。

### 1.2 國際同類服務的最新動向

| 服務 | 最新動態（2025-2026） | 對 i-marathon 的啟示 |
|---|---|---|
| **Marathon Photos Live**（Marathon Photos 旗下，70 國運營） | **讓跑者在比賽當天直接收到照片到手機** | 國際標竿已經在做「即時推播」；我們的 SPEC 方向是對的 |
| **SnapSeek**（Reddit r/SnapSeek 活躍討論中） | 自拍照搜尋（selfie search）為主，主辦方/攝影師工具；500+ 活動案例、ACM 學術論文 SnapSeek 2.0（Video Browser Showdown 2025） | 不對跑者做主動推播；i-marathon 的「主動推播」仍屬差異化 |
| **RaceTagger** | 桌面軟體，賽車/單車專業，98% 準確率 | 桌面工具 vs 我們 serverless 雲端是不同戰場 |
| **Photohawk** | 廠商自述 22% 客戶改提供**免費下載**給跑者（Photohawk FB 貼文） | 國際趨勢可能正在朝「不靠賣照片賺錢」轉向——但此數字未經第三方驗證，僅供方向參考 |
| **mAlbum** | 自拍照一次性交付 | SnapSeek 模式 |

### 1.3 市場驗證結論

✅ **真實需求已被市場反覆驗證**（單場 68 萬張照片規模）。

⚠️ **但純做 SPEC 既有設計「OCR + 自動推播 LINE/IG/Threads」，屬於「比 Sportag/AllSports 多走一步」，並非不可取代的藍海**。

💡 **真正的機會，是把 SPEC 既有設計「做得比國際同類服務更懂女性跑者」——這是一個被國際廠商普遍忽略的 niche**。

## 二、目標受眾：女性跑者的真實輪廓

這不是「跑者」這個泛稱，而是要鎖定一群**有強烈儀式感與分享慾望的女性跑者**：

### 2.1 她是誰？（資料佐證）

- **TS WOMEN RUN（台新女子路跑）**：1.5 萬讚的女性專屬路跑品牌，slogan「**IF YOU HAVE A BODY, YOU ARE AN ATHLETE**」
- **2022 WOMEN RUN TPE**：賽後現場設梳妝編髮、瑜珈伸展、美甲、按摩、冰池——**跑完還是要美美的拍照打卡**
- **名古屋女子馬拉松**：全球女性跑者聖杯，完賽禮是 Tiffany 項鍊 + 完賽披風——**儀式感的極致代表**
- **廣州馬拉松 55 萬粉絲女跑者**：爆紅案例，但「原相踢爆 P 圖」說明**她們強烈渴望「真實美照」，不是過度修圖**
- **Threads @deng1994 的 7 平台整理文**：女性跑者間互相傳閱、分享照片已成為賽後社交儀式

### 2.2 她的核心需求金字塔

```
        ┌─────────────────────┐
        │  紀念價值（實體）    │  ← Phomi 7-11 明信片證明有效
        ├─────────────────────┤
        │  分享即所得（社群）  │  ← IG 限動、Threads、姊妹群組
        ├─────────────────────┤
        │  美感（不是模糊變形）│  ← 美學策展，光線、構圖、情緒
        ├─────────────────────┤
        │  即時（賽事當下）    │  ← Marathon Photos Live 證明可行
        ├─────────────────────┤
        │  真實（不 P 圖）     │  ← 55 萬粉絲案例的反向證據
        └─────────────────────┘
```

### 2.3 為什麼女性跑者這個 niche 重要

1. **付費意願高**：完賽報紙、明信片、客製化相框——她們願意為「儀式感」付費
2. **社群傳播力強**：一張好照片 = 一個 IG 限動 = 觸及 100+ 朋友 = 病毒效應
3. **國際廠商普遍忽略**：SnapSeek/Photohawk 都定位為中性工具，沒針對女性體驗優化
4. **台灣女性路跑賽事蓬勃**：每年 30+ 場女性專屬或女性友好賽事（TS Women Run、WOMEN RUN TPE、台新等）

## 三、三個強化方向（每個都對應 SPEC 既有章節的可強化點）

以下三個方向**都是 SPEC 既有內容的強化**，不是新增主軸功能。三者**可同時並存**，但若要選一個作為第一版 hook：

---

### 🔮 方向 A：「AI 美學策展人」—— 把「識人」升級為「識最美的瞬間」

**對應 SPEC 強化點**：
- §3.2 即時影像處理 → 新增「美學評分管線」（Aesthetic Score Pipeline）
- §3.7 照片歸戶邏輯 → 從「選置信度最高」升級為「選最美的一張」
- §5.2 未來 AI 辨識演進 → 把「姿態美學評分」從未來提案拉到第一版實作

**為什麼這個方向適合女性跑者**：
SPEC §3.7 目前設計是「**OCR 置信度最高的一張**執行推播」。但對女性跑者來說，**置信度高 ≠ 好看**——一張光線糊掉的側面照，OCR 認得號碼布，但這不是她想分享的照片。

**提案內容**：

1. **「美學評分模型」抽象介面**（新增於 §2.10 AI/LLM 抽象層）
   - 在 `IInferenceAdapter` 旁邊新增 `IAestheticScoringAdapter`
   - 評分維度：構圖、光線、人臉表情、肢體動感、色彩氛圍
   - 可用 VLM（GPT-4o / Claude 3.5 Sonnet / Gemini Flash）以 prompt 評分，也可本地開源模型

2. **「最美一張」決策邏輯**（強化 §3.7）
   ```
   叢集內決策函數：
   score_final = α × ocr_confidence 
               + β × aesthetic_score 
               + γ × emotion_score
   其中 α + β + γ = 1，主辦方可調（Feature Flag）
   ```

3. **「推播模板 × 美學評分」組合**
   - 自動選「IG 限動 9:16 最佳構圖」vs「Threads 4:5 最佳構圖」
   - 同一張照片多平台推播時，**針對各平台最佳構圖做智能裁切**

**Hook 故事**：「**不是拍到妳就好，是把妳最美的瞬間找出來**」

**對齊 2026 AI/Agent 敘事**：
- 多模態 VLM 評分（GPT-4o vision, Gemini Flash）
- 與 HKU VideoAgent / Twelve Labs「Video Intelligence is Going Agentic」方向一致（agent 感知+記憶+規劃）

---

### 💌 方向 B：「完賽即所得」—— **v1.0 第一版 hook（核心 + 限定範圍）**

**對應 SPEC 強化點**：
- §3.3 自動社群推播 → 新增「**IG 限動自動模板包**」
- §2.11 功能開關抽象層 → 新增 Feature Flag：`INSTAGRAM_STORY_TEMPLATE`
- §2.8 社群平台整合抽象層 → 推播介面新增「限動尺寸 + 模板化渲染」支援

**為什麼這個方向適合女性跑者**：
SPEC §3.3 目前設計是「推播原圖 + 標準文案」；對女性跑者來說，**一張套好限動模板、配速、完賽時間、賽事 Logo 的 9:16 ready-to-share 圖**，才是「可以一鍵發限動」的真實觸發點。

**提案內容（v1.0 核心範圍）**：

1. **「IG 限動自動模板包」**（新增於 §3.3，純內部實作）
   - 系統預設 12 種限動模板（針對女性跑者設計：櫻花、漸層、馬卡龍、雜誌封面風等）
   - 跑者於註冊時選 2-3 個偏好
   - 推播時直接生成 ready-to-share 限動圖片，含配速、完賽時間、賽事 Logo
   - **零外部依賴**：純內部 Sharp 渲染管線 + 模板檔案管理

**降級為「v1.1+ 外部資源依賴項目」（不列入 v1.0 主軸）**：

2. ⏸️ **「實體明信片兌換券」** — 降級至 v1.1+，需先完成：
   - 與 Phomi 商務合作條款（合作意向書 / 利潤分潤）
   - 印刷廠商之 PDPA 第二類（資料處理者）合約
   - 7-11/全家/萊爾富機台兌換流程（API 串接規格未定）
   - SPEC 先預留 Feature Flag `PHYSICAL_POSTCARD_VOUCHER`，待合作確認後再實作

3. ⏸️ **「姊妹群組推播」機制** — 降級至 v1.2+ 為 P2，需先完成：
   - PDPA 第三人個資合規流程（朋友 LINE ID/IG 帳號需被通知人明確授權，非跑者代為同意）
   - 推播平台之 ToS 審查（避免違反 LINE/Meta 平台規則）
   - SPEC 先預留 Feature Flag `SISTER_GROUP_NOTIFY`，預設關閉

**Hook 故事（v1.0 範圍）**：「**跑完那一刻，妳最美的瞬間已經躺在妳手機裡、妳 IG 限動裡**」

**對齊 2026 AI/Agent 敘事**：
- 「athlete-to-fan」即時 AI 體驗（NBCU × Google Cloud @ 米蘭冬奧）
- Agentic workflow（template selection agent + aesthetic scoring agent + delivery agent）

### 🌸 方向 C：「女性跑者社群第一哩」—— **本版不納，僅作未來討論**

**為什麼本版不納**：

1. **「賽事專屬動態牆」是全新子系統**（會員系統、按讚留言、動態牆 CDN 效能），**違反 SPEC 既有的 serverless 雲端架構定位**。SPEC §2 明確以「無常駐伺服器、API Gateway + Lambda」為核心，新增大規模狀態型子系統會與此架構張力過大。
2. **小紅書從 P2 拉到 P1** 需先解決：小紅書 API 僅開放企業號、繁中內容審查機制不明、跨境個資傳輸法規等議題，**目前證據不足以支持優先級提升**。
3. 如需類似功能對照，**AllSports 已有社群機制**，於 SPEC §1 競品分析中提及即可，無需自建。

**保留為「長期方向」**：方向 C 的精神（女性向 UX、社交裂變、賽事品牌資產）會在 ROADMAP.md（待另開）以「v2.0+」規格記錄，不併入 SPEC.md。

## 五、立即可執行的 SPEC 強化清單

> 以下清單分為 **A. v1.0 可立即合併 SPEC** 與 **B. v1.1+ 預留 Feature Flag** 兩組。
> A 組零外部依賴；B 組需先解決外部資源（商務合作 / 合規流程 / 平台 API）才能實作，SPEC 先預留旗標避免後續破壞性變更。

### A. v1.0 可立即合併 SPEC 的強化點（核心 hook 範圍）

#### 5.1 §3.3 自動社群推播強化（IG 限動模板）
- 新增「IG 限動自動模板包」功能（12 種女性向設計模板）
- 推播介面 `NormalizedPublishTask` 新增 `storyTemplateId?: string` 欄位
- 渲染階段依 `storyTemplateId` 套版（9:16 1080×1920）
- 推播結果 `NormalizedPublishResult` 新增 `renderedStoryUri?: string`（已套模板圖）

#### 5.2 §2.11 功能開關抽象層新增（限動相關）
- 新增 Feature Flag：`INSTAGRAM_STORY_TEMPLATE`（預設開啟，賽事主辦方可關閉）

#### 5.3 §2.8 社群平台整合抽象層強化（限動尺寸支援）
- `ISocialPlatformAdapter` 新增方法：`getStoryTemplateRequirements(platform): StoryTemplateSpec`
- `StoryTemplateSpec` 定義：尺寸、安全區、字型限制（為 IG 9:16 / Threads 9:16 / LINE 9:16 提供對照）
- 推播引擎於渲染階段以 `platform` 為 key 取得 spec，確保各平台限動尺寸正確

### B. v1.1+ 預留 Feature Flag（先占位，後續版本實作）

#### 5.4 預留 Feature Flag（§2.11）
- `PHYSICAL_POSTCARD_VOUCHER`（預設關閉）— 待 Phomi 商務合作 + PDPA 第二類合約確認

- 預留 `IAestheticScoringAdapter` 介面定義（不實作），列入「未來推論引擎矩陣擴充點」

### C. 不納項目（明確排除）

- ❌ 「賽事專屬動態牆」—— 屬全新子系統，違反 SPEC serverless 架構定位，本版不納

## 六、第一版 hook 的具體故事（給行銷/業務看，限縮後版）

### 🎬 故事標題：「**跑完那一刻，妳已經在 IG 限動裡了**」

**使用者旅程（v1.0 範圍）**：
1. 小美報名「2026 台新女子路跑 10K」，註冊時勾選 2 個偏好限動模板（「櫻花漸層」+「雜誌封面」）
2. 跑到 8K 處被攝影師拍下（不動聲色，她甚至沒注意到）
3. 完賽衝線 30 秒後，LINE 收到通知：「**小美，妳最美的完賽照在這裡 ✨**」
4. 點開是已經套好「櫻花漸層模板」+ 配速 5'32"/km + 完賽時間 55:23 + 賽事 Logo 的 9:16 ready-to-share 限動圖
5. 一鍵分享到 IG 限動

**為什麼這個故事會被分享**：
- 「**跑完就能立刻拿到套好模板的美照**」是其他平台都做不到的（AllSports 要等 3-7 天、Sportag 要自己上傳自拍照搜尋）
